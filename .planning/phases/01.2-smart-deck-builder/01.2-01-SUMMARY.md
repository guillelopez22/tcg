# Phase 01.2 Plan 01: DB Migration and Type Contracts Summary

**Zone-aware deck data layer: added zone/status/keywords columns, updated unique index, Riftbound copy limits (MAX_COPIES=3), and pure shared validateDeckFormat utility**

## Accomplishments
- DB schema updated with zone on deck_cards, status on decks, keywords on cards; unique index changed to (deck_id, card_id, zone)
- All Riftbound deck limits codified as named constants: MAX_COPIES_PER_CARD=3, MAIN_DECK_SIZE=40, RUNE_DECK_SIZE=12, CHAMPION_COUNT=1, SIDEBOARD_SIZE=8
- Pure validateDeckFormat function checks all 6 format rules (zone totals, copy limits, signature limits) with no platform dependencies
- Zone-aware deckCardEntrySchema, new deckSuggestSchema and deckBuildabilitySchema exported from shared package

## Decisions Made
- Zone as third component of unique index: same card can appear in both main and sideboard (required by Riftbound rules)
- validateDeckFormat returns string[] (error messages); empty array = valid deck
- Sideboard is 0 or exactly 8 — partial sideboards are invalid per Riftbound format
- MAX_COPIES_PER_CARD imported from shared in deck.service.ts, not redeclared
