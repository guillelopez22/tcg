## Phase Boundary

Upgrade the existing deck builder into a smart, collection-aware editor with format validation (40 main + 12 runes + 1 champion + sideboard), real-time analytics (energy curve, domain distribution, card type breakdown, rarity, market value), ownership integration (own badges, buildability %, wantlist integration), and intelligent card suggestions (synergy-based with domain + curve + meta co-occurrence + text keyword analysis). <decisions>
## Implementation Decisions

### Format Enforcement
- Deck zones presented as three tabs in the deck panel: Main (40) | Runes (12) | Champion (1) — plus a Sideboard tab
- Cards auto-zone by card type: Legends go to Champion, Rune-type cards to Runes, everything else to Main
- When a zone is full, block the add and show inline swap prompt: list existing cards in that zone with 'Replace?' buttons
- Champion-first workflow enforced for Legend and Suggested paths only; Scratch path allows adding champion whenever
- Persistent validity status bar at top of editor: green "Valid" or red list of issues ("Need 1 champion", "Main: 35/40")
- Card search panel auto-filters based on active zone tab (Runes tab = only rune-type cards shown, Main tab = hides legends and runes)
- Per-card copy limits follow Riftbound official game rules (Claude to research exact limits per card type)
- Sideboard zone for competitive play (Claude to research official Riftbound sideboard size; if no official rule, use 15-card convention)
- Saving allowed even with invalid/incomplete decks: deck marked as "Draft" (invalid) or "Complete" (valid), status shown in deck list

### Collection Integration
- Each card in search results shows ownership count badge ("Own: 3" or "Own: 0" in muted)
- Adding an unowned card to deck: allow freely but show subtle prompt "You don't own this card. Add to wantlist?" with yes/no
- Buildability percentage shown on both deck list page AND deck detail page ("You own 32/53 cards (60%)")
- Deck detail page has one-tap "Add X missing to wantlist" button (reuses existing wantlist add pattern)

### Card Suggestion Engine
- Suggestions appear both at deck creation (auto-fill) AND during editing (real-time in editor)
- Server-side tRPC endpoint: takes current deck cards, returns ranked suggestions (10-15 cards)
- Suggestion signals: domain match + energy curve balance + meta co-occurrence (trending/tournament deck data) + text keyword synergy
- Keyword synergy uses curated keyword tags on cards (sacrifice, draw, buff, heal, etc.) with a periodic cron service to re-tag when card data updates (follows existing deck-sync cron pattern)
- Suggestions displayed in a "Suggested" tab within the search panel (alongside "Search" tab)
- Each suggestion shows a brief reason tag ("Meta pick", "Curve filler", "Synergy: sacrifice") with expandable detail on tap ("Appears in 4 S-tier decks with your champion, fills 3-cost gap")
- Toggle between "Owned first" and "Best fit" ranking in suggestions panel
- Suggestions refresh via manual "Refresh suggestions" button (not automatic on every deck change)
- No cut/remove suggestions — only suggest cards to add
- At deck creation (legend path): offer two options — "Build with my cards" (prefer owned) and "Build best deck" (ignore collection)
- Auto-complete button in editor prompts user before filling: "Fill 8 remaining main deck slots with suggested cards?" with preview of what would be added

### Deck Analytics
- Mini charts in editor: compact energy curve bar chart + domain distribution mini-pie, updating in real-time as cards change
- Full analytics in separate tab/section on deck detail page with: energy curve, domain distribution, card type breakdown, rarity distribution, total deck market value
- All analytics compute client-side from card attributes already loaded (no extra API call for charts)
- recharts library (already used in collection stats) for chart rendering

### Claude's Discretion
- Exact synergy keyword taxonomy and tagging approach
- Riftbound-specific copy limits per card type (research game rules)
- Sideboard size (research official rules, default to 15)
- Mini chart exact styling and responsive layout within the tabbed editor
- Auto-complete preview UI design
- Suggestion endpoint caching strategy
- Swap prompt UI layout and interaction details

</decisions>

<specifics>
## Specific Ideas

- Auto-complete should preview what would be added before the user confirms — not a blind fill
- Legend-path creation offers "Build with my cards" vs "Build best deck" as two explicit buttons before auto-fill runs
- Suggestion reason tags should feel informative not overwhelming — brief tag visible, detail only on tap
- The periodic keyword tagging service follows the same cron pattern as the existing deck-sync module (@nestjs/schedule)
- Format validity status bar should feel like a linter — always present, green when valid, red with specific issues when not
- Buildability % on the deck list motivates completing decks and drives wantlist usage

</specifics>
