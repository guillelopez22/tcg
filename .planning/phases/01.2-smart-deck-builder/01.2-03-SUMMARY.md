## Decisions Made

- `validateDeckFormat` called client-side on every deck state change — exact same function as server, eliminates any divergence risk on format rules
- Ownership data sourced via `collection.list(limit:200)` bulk query into a `Map<cardId, count>` — single query, no per-card requests
- Swap prompt is an inline list with per-card Replace buttons — shown when zone-full add is attempted, more actionable than silent disable
- Auto-complete button only enables when the Suggestions panel is active — user sees what will be added before triggering
- `choose-build-mode` wizard step navigates directly to `name-deck` on button click, not a separate confirm step
