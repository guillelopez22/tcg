  <action>
Refactor `deck-card-editor.tsx` into a zone-aware editor. This is the primary refactor target of the phase.

**State management:**
- `activeZone: DeckZone` state (default 'main') -- controls which zone tab is active
- `searchPanel: 'search' | 'suggested'` state -- toggles between search and suggestions in left panel
- `deckEntries: Map<string, {cardId, quantity, zone, card}>` -- local working copy of deck cards, grouped by zone
- `ownershipMap: Map<string, number>` -- card ownership counts, loaded via a bulk query

**Zone tabs (right panel header):**
- Four tabs: Main ({count}/40) | Runes ({count}/12) | Champion ({count}/1) | Sideboard ({count}/8)
- Active tab highlighted. Counts update in real-time as cards are added/removed.
- Use `lg-*` design system classes for tab styling.

**Validity status bar (top of editor, persistent):**
- Compute validation client-side using the same rules as server: main=40, rune=12, champion=1, sideboard=0 or 8, 3-copy limit, 1-copy signature limit
- Green bar with checkmark icon + "Valid" when all rules pass
- Red bar with list of issues when rules fail (e.g., "Need 1 champion", "Main: 35/40", "Sideboard: 5 (must be 0 or 8)")
- Import zone size constants from `@la-grieta/shared`

**Card search auto-filtering (left panel):**
- When active zone is 'main': filter search results to exclude cardType 'Legend' and 'Rune'
- When active zone is 'rune': filter to only show cardType 'Rune'
- When active zone is 'champion': filter to only show cardType 'Legend'
- When active zone is 'sideboard': same as main (exclude Legend, Rune)
- Apply filter as additional parameter to the card.list tRPC query OR filter client-side from results

**Ownership badges:**
- On component mount (or when deck loads), call a bulk query to get ownership counts for all cards visible in search results. Use `trpc.collection.list` or add a new lightweight endpoint. Alternatively, load the user's full collection card IDs + counts in one query and store in a Map.
- Each card in search results shows "Own: {count}" badge. Count > 0 in accent color (rift-400), count === 0 in muted (zinc-600).

**Wantlist prompt on unowned card add:**
- When user adds a card with ownership count 0 to the deck, show sonner toast: "You don't own this card." with action button "Add to wantlist"
- On click, call existing `wishlist.toggle` mutation with type 'want'

**Swap prompt when zone full:**
- When user tries to add a card to a zone that's at capacity (e.g., main at 40), block the add
- Show an inline prompt below the card list in that zone: "Main deck is full. Replace a card?" with a list of existing cards in that zone, each with a "Replace" button
- Clicking "Replace" removes the old card and adds the new one
- Dismiss the prompt with an X button

**Save button:**
- Collects all entries from all zones into a flat array with `{cardId, quantity, zone}`
- Calls `setCards` mutation
- On success, invalidates getById query

**Per-card copy limits:**
- Prevent adding more than 3 copies of any card (across main + sideboard)
- Prevent adding more than 1 copy of Signature cards (check cardType against SIGNATURE_TYPES)
- Show toast error when limit reached
