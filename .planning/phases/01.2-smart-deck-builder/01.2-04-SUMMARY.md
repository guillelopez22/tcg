## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] computeAnalytics missing from @la-grieta/shared index exports**
- **Found during:** Task 2 (creating deck-analytics.tsx importing from @la-grieta/shared)
- **Issue:** `packages/shared/src/utils/compute-analytics.ts` existed since Plan 02 but was never added to `packages/shared/src/index.ts` or `index.js`, so `import { computeAnalytics } from '@la-grieta/shared'` would resolve to undefined
- **Fix:** Added `export * from './utils/compute-analytics'` to both `index.ts` and `index.js`
- **Files modified:** packages/shared/src/index.ts, packages/shared/src/index.js
- **Verification:** `npx tsc --noEmit -p apps/web/tsconfig.json` passes
- **Committed in:** 6ab8616 (Task 2 commit)

**2. [Rule 1 - Bug] DeckCardWithCard type missing energyCost field**
- **Found during:** Task 2 (deck-analytics.tsx passes cards to computeAnalytics which needs energyCost)
- **Issue:** `DeckCardWithCard.card` type didn't include `energyCost`, and the `getById` DB select also didn't include `cards.energyCost`. Without this, energy curve chart would always show 0 cost.
- **Fix:** Added `energyCost: number | null` to type definition and `energyCost: cards.energyCost` to DB select
- **Files modified:** apps/api/src/modules/deck/deck.service.ts
- **Verification:** `npx tsc --noEmit -p apps/api/tsconfig.json` passes
- **Committed in:** 6ab8616 (Task 2 commit)

**3. [Rule 1 - Bug] deck-wizard.tsx stepTitle Record missing 'choose-build-mode' key**
- **Found during:** Task 2 (TypeScript compilation check surfaced this pre-existing error)
- **Issue:** `WizardStep` union type included `'choose-build-mode'` but `stepTitle: Record<WizardStep, string>` was missing this key, causing TS2741
- **Fix:** Added `'choose-build-mode': 'Build Options'` to the stepTitle record
- **Files modified:** apps/web/src/app/(dashboard)/decks/deck-wizard.tsx
- **Verification:** TypeScript compilation passes after fix
- **Committed in:** 6ab8616 (Task 2 commit)
