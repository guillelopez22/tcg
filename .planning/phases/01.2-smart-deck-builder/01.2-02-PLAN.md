   - Input: `{ deckId, mode: 'owned_first' | 'best_fit', zone }` (from DeckSuggestInput)
   - Inject Redis client (same pattern as DeckRecommendationsService: `@Inject('REDIS_CLIENT')`)
   - Cache key: `deck:suggest:${deckId}:${mode}:${zone}`, TTL 10 minutes
   - Logic:
     a. Load current deck cards to understand what's already in the deck
     b. Determine what's needed: if zone is 'main', find cards that are not Legend or Rune type
     c. Load candidate cards from DB (filter by appropriate card types for zone)
     d. For each candidate, compute score:

        - Curve balance: +2 if card fills a gap in the energy curve (energyCost bucket with fewest cards)
        - Co-occurrence: +2 if card appears in trending/public decks that share the same champion
        - Keyword synergy: +1 per matching keyword between card and existing deck cards
     e. If mode === 'owned_first': boost owned cards by +5
     f. Sort by score descending, take top 15
     g. Generate reason tag for each: pick highest-scoring signal ("Meta pick", "Curve filler", "Synergy: {keyword}", "Domain match", "Owned")
     h. Return `SuggestedCard[]` with `{ cardId, card: {...}, score, reasonTag, reasonDetail, owned: boolean }`
