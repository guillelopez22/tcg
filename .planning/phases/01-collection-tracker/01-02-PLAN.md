    - addBulk(userId, [{cardId:'a'}, {cardId:'b'}]) creates one row per entry (not per quantity)
    - addBulk with >50 entries throws BAD_REQUEST
    - update(userId, {id, variant:'alt_art'}) updates the variant on that specific copy
    - update(userId, {id, purchasePrice:'12.50'}) stores the purchase price
    - update(userId, {id, photoUrl, photoKey}) stores the photo reference
    - remove(userId, {id}) deletes exactly that one copy row
    - remove(userId, {id}) where id belongs to another user throws NOT_FOUND
    - list(userId, {}) returns cards grouped by cardId with copyCount, sorted by date_added desc
    - list(userId, {variant:'alt_art'}) filters to only cards where user has alt_art copies
    - list(userId, {sortBy:'name', sortDir:'asc'}) sorts alphabetically by card name
    - getByCard(userId, {cardId}) returns all copies of that card for that user
    - getUploadUrl({contentType:'image/jpeg'}) returns presigned URL from R2 with collection purpose
  </behavior>
  <action>

       - Write tests for each behavior listed above
       - Ensure add() asserts a new row is created each time (not upserted)
       - Test that listing aggregates by cardId with copy count

    2. Rework `collection.service.ts`:
       - `add()`: REMOVE the existing check-and-upsert logic entirely. Always `db.insert(collections).values({...}).returning()`. Default variant to 'normal', condition to 'near_mint'.
       - `addBulk()`: For each entry, insert a single row. Do NOT multiply by quantity (quantity is gone). Use `db.insert(collections).values(entries.map(...)).returning()`.
       - `update()`: Accept id + optional fields (variant, condition, purchasePrice, notes, photoUrl, photoKey). Only update provided fields. Verify ownership (userId match) before updating.
       - `remove()`: Delete by id, verify ownership first.
       - `list()`: Query collections grouped by cardId. Return: `{ cardId, card: {...}, copyCount, variants: string[], latestCondition, latestUpdatedAt }`. Apply filters (set, rarity, variant, condition, domain) and sorting (name, date_added, price, set_number). Use cursor pagination with `buildPaginatedResult` from shared.
       - `getByCard()`: Return all collection rows for a userId+cardId, joined with card data.
       - `getUploadUrl()`: Call R2 `generateUploadUrl` with purpose 'collection', return `{uploadUrl, publicUrl, key}`.

    3. Update `collection.router.ts`:
       - Ensure all procedures use the updated Zod schemas from Plan 01
       - Add `getByCard` protected procedure with `collectionGetByCardSchema` input
       - Add `getUploadUrl` protected procedure accepting `{contentType: z.string()}`
       - Remove any procedures that reference `quantity`

    CRITICAL: The old upsert logic (check existing + increment quantity) MUST be completely removed. Every `add()` call creates a new row. This is the core schema model change.
