    2. Rework `card-scanner.tsx`:
       - Maintain the existing getUserMedia camera feed (works on both desktop webcam and mobile via facingMode: 'environment')
       - Add continuous auto-scan loop: capture frame every ~1 second, send to scanner.identify
       - Confidence threshold: only trigger detection when NCC score >= 0.93 (raw). This maps to 90%+ normalized confidence per RESEARCH.md formula: `displayPct = Math.round(((score - NCC_THRESHOLD) / (1 - NCC_THRESHOLD)) * 100)` where NCC_THRESHOLD = 0.3.
       - When detection triggers: pause scanning, show ScanConfirmation overlay
       - Session state (React state, not persisted): array of `{card, variant, condition, quantity, addedAt}` entries
       - Session counter badge: fixed overlay on camera feed showing count of cards added this session
       - All strings via `useTranslations('scanner')`
       - Add viewfinder frame overlay on camera feed (simple border/bracket design — Claude's discretion on exact style)

    3. Create `scan-confirmation.tsx`:
       - Overlay shown when a card is detected at 90%+ confidence
       - Shows: card art (large), card name, confidence percentage (normalized to 0-100%), set name
       - Quantity +/- buttons (default 1, range 1-20) — each creates that many copies
       - Variant toggle: 4 buttons for Normal/Alt-Art/Overnumbered/Signature
         - For now, all variants enabled for all cards (see RESEARCH.md open question about card-specific validation)
         - Style unselected variants as muted but clickable
       - Condition defaults to NM, but show dropdown to change
       - "Add" button: calls `trpc.collection.addBulk.useMutation` with quantity copies at selected variant/condition
         - On success: show success toast via sonner, add to session state, auto-close confirmation
         - After close: wait cooldown duration, then resume scanning
       - "Skip" button: close confirmation without adding, resume scanning immediately

    4. Create `scanner-settings.tsx`:
       - Small settings dropdown accessible from scanner page header
       - Cooldown duration: dropdown with options [1s, 2s, 3s, 5s, 10s] — default 3 seconds (Claude's discretion per CONTEXT.md)
       - Store in localStorage so it persists across sessions
       - Settings icon in scanner page header bar

    5. Update `page.tsx` to compose the scanner page:
       - Camera feed (card-scanner) as main content
       - Session counter badge overlaid
       - "End Session" button that navigates to scan session summary
       - Settings button in header

    CRITICAL: Scanner MUST NOT auto-add cards. The user MUST tap "Add" on the confirmation overlay. This is a locked decision.
