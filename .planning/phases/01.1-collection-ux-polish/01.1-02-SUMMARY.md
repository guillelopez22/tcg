## Decisions Made

- **Optimistic delta map instead of cache setData:** `Map<cardId, number>` state tracks local deltas separately from tRPC's infinite query cache. This avoids the complexity of `setData` on paginated infinite queries while still delivering instant UI feedback. Delta is cleared on `onSettled` and the cache invalidates normally.
- **Undo captures copy before mutation:** The removed copy's `{ id, cardId, variant, condition }` is captured before calling `removeMutation.mutate`. The undo action re-adds with those exact values so the re-added copy matches what was removed.
- **md: breakpoint for hover steppers (not sm:):** The plan specified `md:` which is 768px. At sm: (640px) the device is still likely a phone in landscape. Using `md:` as the desktop threshold is correct.
- **Inline absolute popovers:** Variant picker and copy picker render as absolute children of the card tile with a fixed-position backdrop for click-away. No portal needed at this scale.
- **Tasks 1 and 2 in one commit:** Both tasks modify only `collection-grid.tsx`. Writing them separately would have left Task 1 in a broken intermediate state (no long-press on the + button). Single atomic write documented as deviation.
