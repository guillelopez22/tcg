## Decisions Made
- Hover reveal uses `md:opacity-0 md:group-hover:opacity-100` (md breakpoint, not sm) — mobile-first: always visible on phones and small tablets
- `buildStarterCardsFromPool` still uses 6 chained `card.list` queries for the full card pool; the legend picker itself uses `card.legends` endpoint (plan requirement honored)
- Preview step uses `trpc.deck.getById.useQuery` (reactive) rather than imperative fetch, enabled only when `selectedDeckId` is set and `step === 'preview-suggested'`
- Undo toast calls `toast.dismiss(toastId)` when undo is clicked to remove it immediately
