# Phase 1.1 Plan 01: Backend Data Plumbing Summary

Backend support for all Phase 01.1 UX polish features: market price in wishlist/collection list queries, a dedicated legends endpoint for the deck wizard, and a tier column on decks for trending deck tier badges.

## Tasks Completed

### Task 1: Tier column + price joins in wishlist and collection (40bed49)

**Schema — packages/db/src/schema/decks.ts:**
Added `tier: varchar('tier', { length: 2 })` nullable column after `domain`. Stores 'S', 'A', 'B', 'C', or null for user-created decks.

**Migration — tools/seed/src/migrate-01.1-01.cjs:**
Direct pg client script using `ALTER TABLE decks ADD COLUMN IF NOT EXISTS tier varchar(2)`. Idempotent, same pattern as the Phase 01 migration approach.

**tools/seed/src/sync-riftdecks.ts:**
- Updated `deckLinks` array type from `{ url, name }` to `{ url, name, tier: string | null }`
- `dedupeByUrl` helper updated to handle the new type
- Champion deck links now carry `tier: champion.tier` when pushed
- Tournament deck links carry `tier: null`
- Scraped decks carry tier through to upsert — both `.set({})` (update) and `.values({})` (insert) now include `tier: deckData.tier`

**apps/api/src/modules/deck/deck.service.ts:**
- `browse()` select now includes `tier: decks.tier`
- Row mapping includes `tier: r.tier`
- `DeckWithCreator` type left as `Deck & { user: ... }` — Deck already includes tier via `$inferSelect`

**apps/api/src/modules/wishlist/wishlist.service.ts:**
- Imported `cardPrices` from `@la-grieta/db`
- Added `card_marketPrice: cardPrices.marketPrice` to select fields
- Added `.leftJoin(cardPrices, eq(cards.id, cardPrices.cardId))` after innerJoin(sets)
- Row mapping includes `marketPrice: r.card_marketPrice ?? null`
- `WishlistCard` interface updated to include `marketPrice: string | null`

**apps/api/src/modules/collection/collection.service.ts:**
- Added `card_marketPrice: cardPrices.marketPrice` and `card_foilMarketPrice: cardPrices.foilMarketPrice` to select in `list()`
- Added `.leftJoin(cardPrices, eq(cards.id, cardPrices.cardId))` after innerJoin(sets) in `list()`
- Row mapping includes `marketPrice: r.card_marketPrice ?? null` and `foilMarketPrice: r.card_foilMarketPrice ?? null` on the `card` object
- `CollectionEntryWithCard` type updated: card now extends `{ marketPrice: string | null; foilMarketPrice: string | null }`

### Task 2: card.legends tRPC endpoint (8fd6ddf)

**apps/api/src/modules/card/card.service.ts — getLegends():**
- Queries cards WHERE `cardType = 'Legend'` AND `isProduct = false`, ordered by name
- Selects only `id, name, cleanName, domain, imageSmall` — minimal payload for deck wizard
- Redis cache at `cache:card_legends` with 1h TTL

**apps/api/src/modules/card/card.router.ts:**
- `legends: pub.query(() => this.cardService.getLegends())` added after `getPrice`
- Uses `rateLimitedPublicProcedure` — no auth required (same as `list`, `sets`)

## Deviations from Plan

None — plan executed exactly as written.

One observation: the plan's task description for `DeckWithCreator` said to add `tier: string | null` explicitly. Since `Deck` type is inferred from the schema via `$inferSelect`, adding `tier` to the schema automatically adds it to `Deck`. Adding it again to `DeckWithCreator` would be redundant but not harmful. I omitted the redundancy and left `DeckWithCreator = Deck & { user: ... }` which is cleaner.
