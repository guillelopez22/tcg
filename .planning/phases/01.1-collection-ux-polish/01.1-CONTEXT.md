# Phase 01.1: Collection UX Polish - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Polish the UX of four features added in Phase 1's final commit: collection grid +/- steppers, tradelist asking price editor, wantlist/tradelist remove buttons, and deck wizard. All features exist and function — this phase improves interactions, feedback, and visual consistency. No new capabilities.

</domain>

<decisions>
## Implementation Decisions

### Collection Grid Interactions
- Tap - removes last-added copy immediately (no confirmation dialog)
- Undo via 5-second auto-dismiss sonner toast with "Undo" action button
- When count reaches 0, card stays in grid with "0" badge for a few seconds, then auto-removes on next refetch
- +/- steppers always visible on mobile (current behavior), hover-reveal on desktop
- Optimistic UI: count bumps instantly on tap, rolls back if API fails
- Tap + adds Normal/NM instantly. Long-press + reveals variant/condition picker with only valid variants for that card (hide/gray invalid ones)
- When tapping - on a card with 2+ copies, show a mini picker to choose which copy to remove (condition badges)
- No card name label below grid cards — image only (current behavior)
- Show "Total: $X" badge in filter/sort bar area, updates as filters change

### Tradelist Pricing Experience
- Price editor pre-fills with TCGplayer market price from card_prices table
- Action buttons (price, remove) appear on hover on desktop, always visible at card bottom on mobile
- Inline price editor stays at card bottom (no bottom sheet)
- Per-card pricing only — no batch "price all" action
- Show both asking price and market price on each tradelist card (e.g., "Ask: $5.00 | Mkt: $4.50")

### Wantlist/Tradelist Management
- Show market price on each wantlist card plus "Total want value: $X" summary at top
- Keep as simple lists — no sort/filter options
- All remove actions across collection, wantlist, tradelist get consistent 5-second undo toast
- Visibility toggle bulk update approach: Claude's discretion (single bulk API vs sequential)

### Deck Wizard Refinements
- Card loading optimization: Claude's discretion (dedicated legends endpoint vs current chained queries)
- Trending deck import shows card list preview with counts (grouped by type: Legend, Creatures, Runes) before confirming import
- Show tier badge (S/A/B/C from riftdecks) next to each trending deck
- Legend selection step has search bar + domain filter chips (Fury, Calm, Mind, etc.)
- No ownership count on legend selection — keep it simple
- After creating deck from legend path, show summary ("Created! 40 cards added" with domain breakdown) before "Go to Editor" button
- Non-English deck names: try to translate to English, fall back to legend name + domain if unclear, show original script as subtitle
- Deck name display: translate Chinese/non-Latin names, show original in parentheses or subtitle

### Claude's Discretion
- Copy picker UI style (inline dropdown vs popover) when removing from multi-copy cards
- Bulk visibility toggle implementation (single API endpoint vs sequential)
- Card loading optimization for wizard (dedicated endpoint vs current approach)
- Exact long-press duration threshold for variant picker
- Translation approach for non-English deck names (client-side detection vs server-side processing)

</decisions>

<specifics>
## Specific Ideas

- Long-press on + for variant picker is a power-user shortcut — normal tap should feel instant and frictionless
- Tradelist should show the spread between asking and market price so users can see if their price is competitive
- Undo toast pattern should be consistent across all destructive actions (remove copy, remove from wantlist, remove from tradelist)
- Many riftdecks.com trending decks have Chinese names — translation/fallback is critical for the Honduran user base
- Deck preview before import helps users avoid importing decks they don't want and having to delete them

</specifics>
