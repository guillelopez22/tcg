    2. Create `apps/api/src/modules/match/match.service.ts`:
    - Injectable service with db (Drizzle), redis (ioredis) injected via constructor.
    - `create(userId: string | null, input: MatchCreateInput)`: Generate 6-char nanoid code. Insert into match_sessions with status 'waiting', create match_players rows for initial players with auto-assigned colors from PLAYER_COLORS. Call createInitialState() from match-scoring. Store state in jsonb column. Return { code, sessionId }.
    - `join(userId: string | null, input: MatchJoinInput)`: Look up session by code. If status is 'waiting' or 'active', insert match_player row with next available color. Return session state. If not found, throw NOT_FOUND.
    - `getFullState(code: string)`: Query match_sessions by code, return parsed state jsonb + player list.
    - `getById(sessionId: string)`: Query match_sessions by UUID id (not code). Return full match data including state JSON, player list, format, status, startedAt, endedAt, winnerId. Used by match history detail view (Plan 05). Input validated as z.string().uuid().
    - `applyBattlefieldTap(code: string, data: { battlefieldIndex, playerId })`: Load state, call cycleBattlefieldControl, check if conquest (control changed to a player), if so call scoreConquest. Check validateWinCondition. Save updated state to DB. Return new state.
    - `advancePhase(code: string, playerId: string)`: Load state, call advancePhase. If entering 'B' phase, call scoreBeginningPhase. Save state. Return new state.
    - `advanceTurn(code: string)`: Load state, call advanceTurn from scoring engine. Save. Return.
    - `pauseMatch(code: string)`: Toggle isPaused in state. Save. Return.
    - `endMatch(code: string, winnerId: string | null, reason: 'score' | 'concession')`: Set status='completed', endedAt=now, winnerId. Update match_players with finalScore and isWinner. Return summary.
    - `submitBattlefieldSelection(code: string, playerId: string, cardIds: string[])`: Store the player's secret battlefield card selections in a pending map on the session state (`state.pendingBattlefieldSelections[playerId] = cardIds`). Do NOT broadcast the individual selection to other players -- only store it server-side. Check if all players (role='player') have submitted. Return `{ allSubmitted: boolean }`.
    - `revealBattlefields(code: string)`: Called when all players have submitted battlefield selections. Move `pendingBattlefieldSelections` into the active battlefield slots (`state.battlefields[i].cardId`, `.cardName`, `.cardArt` populated by looking up card data from the DB via card IDs). Clear `pendingBattlefieldSelections`. Set session status to 'active' if it was 'waiting'. Return the fully revealed state with battlefield card art and names.
    - `history(userId: string, input: MatchHistoryInput)`: Query match_sessions joined with match_players where match_players.userId = userId. Order by endedAt desc. Cursor pagination. Return list with format, players, scores, winner, date, duration.
    - All state mutations use UPDATE ... WHERE code = ... RETURNING to prevent race conditions.

    3. Create `apps/api/src/modules/match/match.router.ts`:
    - Follow existing router pattern (deck.router.ts).
    - `create`: optionalAuthProcedure, input(matchCreateSchema), mutation. Guests can create matches.
    - `join`: optionalAuthProcedure, input(matchJoinSchema), mutation.
    - `getState`: publicProcedure, input(z.object({ code: z.string() })), query. For spectators.
    - `getById`: protectedProcedure, input(z.object({ id: z.string().uuid() })), query. For match history detail view. Returns full match data including state JSON.
    - `history`: rateLimitedProtectedProcedure, input(matchHistorySchema), query.
    - Register router in buildRouter() returning `this.trpc.router({ ... })`.

    4. Create `apps/api/src/modules/match/match.gateway.ts`:
    - `@WebSocketGateway({ cors: { origin: process.env['CORS_ORIGINS']?.split(',') ?? ['http://localhost:3000'], credentials: true }, namespace: '/match' })`
    - Inject MatchService.
    - `handleConnection(client)`: Extract code from handshake query. Join room by code. Send full state via `state:full` event.
    - `handleDisconnect(client)`: Broadcast `player:disconnected` to room.
    - `@SubscribeMessage('battlefield:tap')`: Call matchService.applyBattlefieldTap, broadcast `state:patch` to room.
    - `@SubscribeMessage('battlefield:submit')`: Call matchService.submitBattlefieldSelection(code, playerId, cardIds). Emit `battlefield:submitted` back to ONLY the submitting client (acknowledgment). If allSubmitted=true, call matchService.revealBattlefields(code) and broadcast `battlefield:reveal` with the revealed state to the ENTIRE room. This implements the "secret pick + simultaneous reveal" per user decision.
    - `@SubscribeMessage('phase:advance')`: Call matchService.advancePhase, broadcast.
    - `@SubscribeMessage('turn:advance')`: Call matchService.advanceTurn, broadcast.
    - `@SubscribeMessage('match:pause')`: Call matchService.pauseMatch, broadcast.
    - `@SubscribeMessage('match:end')`: Call matchService.endMatch, broadcast `match:ended`.
    - `@SubscribeMessage('match:undo')`: Restore previous state (keep one-deep undo in state as `previousBattlefields`).
    - Configure 15-second heartbeat: `pingInterval: 15000, pingTimeout: 30000` in gateway decorator options.
